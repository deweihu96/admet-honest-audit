Architecture

One loop that climbs — with the test set walled off.

Four roles, separated so the component that can see the test set cannot influence the component that designs models. Honesty is enforced by what each module can import, not by good intentions.

Iteration loop · validation only
● Agent · Proposer

Proposer

Reasons about validation results and designs the next model version.

Claude Code · sees validation only
● Agent · Executor

Executor

Trains each candidate and scores it on validation across seeds.

no test path, by construction
▤ Deterministic · Arbiter

Arbiter

Promotes a model only on non-overlapping confidence intervals, keeps the simpler model on ties, and detects plateaus. Pure code — it can't be argued into a bad promotion.

no LLM · fully logged
◆ Walled · Auditor

Auditor

The only component allowed to touch test-set molecules — for leakage checks. Returns a summary verdict with no labels, no per-molecule data.

two-axis: Tanimoto + InChIKey
◈ Human-in-the-loop

You

Inject an idea the agent didn't reach; command when to test. Held to the same discipline — repeated tests widen the reported uncertainty.

no privilege · test reads counted
Test wall

The test set is reached only by the auditor (molecules, never labels) and by a single final evaluation, run once after a model is locked. The iteration loop has no code path to it.

candidate→ train→ score on validation→ arbiter: promote / tie→ lock→ test · once→ bootstrap CI + verdict
validation loop (agent works here) test boundary (auditor + final eval only) human, same discipline as the agent
