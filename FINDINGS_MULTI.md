(classifier specs for classification, LightGBM-based regression specs for
regression). Test touched once per endpoint via the normal user-commanded read.

## The three questions, answered with the aggregated evidence

### Q1 — How often does honest iteration CI-separate models vs plateau?
**3 of 10 separate (30%); 7 of 10 plateau (70%).** Every separation is the same
move — descriptors CI-separably beating fingerprints on a physicochemistry-driven
regression (solubility, caco2, lipophilicity: all logP / permeability / solubility
tasks where a descriptor like cLogP *is* the label's main driver). Everywhere
else the candidate stream's models are statistically indistinguishable: the small
imbalanced classification tasks (cyp2d6, cyp2c9, dili, bbb), the rank-correlation
clearance task, ld50, and even a mid-size regression (ppbr) where fingerprints
and descriptors tie. Takeaway: real *climbable, separable* signal within a modest
candidate stream is the exception, not the rule — honest iteration mostly and
correctly reports "these models can't be told apart," which is exactly the
audit-literature expectation. Where it does climb, it's a physically-motivated
representation change, not hyperparameter luck.

### Q2 — How does bootstrap CI width track test-set size?
The sampling-error story holds most cleanly **within a metric family**, and is
starkest for the small-classification / PR-AUC regime:
- **PR-AUC, ~135 test (~35–40 minority positives): width ≈ 0.25–0.29** (cyp2d6
  0.252, cyp2c9 0.288). These intervals span most of a leaderboard — fine
  rankings there are pure sampling noise.
- **ROC-AUC**: dili (96 test) 0.094 and bbb (406) 0.076 — tighter, partly because
  the scores sit near the 0.84–0.93 ceiling where AUROC compresses.
- **Spearman, 243 test: 0.216** — rank correlation on a few hundred points is
  wide.
- **log-scale MAE**: caco2 (182) 0.061, lipophilicity (840) 0.063, ld50 (1478)
  0.062, solubility (1997) 0.062 — nearly constant, because MAE CI scales with
  `label_spread / sqrt(n_test)`, and the smaller sets here have narrower label
  distributions that offset their size. So raw test size alone does NOT predict
  MAE width; label spread confounds it.

Conclusion: **small endpoints → wide CI → rankings within noise is confirmed and
dramatic for the PR-AUC/small-classification regime** (the exact regime the
Receptor.AI audit flagged), while for MAE the interval width is governed by
label spread as much as by test size. Either way, the tightest intervals belong
to the large, well-spread endpoints.

### Q3 — How many endpoints show leakage or #217 version-drift?
**2 of 10 endpoints carry real train/test molecular identity leakage:**
- **solubility_aqsoldb: 7** InChIKey-identity duplicates (flagged).
- **bbb_martins: 2** InChIKey-identity duplicates (flagged) — same-drug/different-
  salt pairs the Tanimoto axis (max 0.837) missed entirely.

**Issue-#217 status across the 3 documented targets checked — silent version
drift, both directions:**
- **caco2_wang: PATCHED** (InChIKey 0, NN max 0.953).
- **ld50_zhu: PATCHED** (InChIKey 0; the NN-max 1.000 is a fingerprint collision,
  not a duplicate).
- **bbb_martins: STILL PRESENT** (InChIKey 2).

So a documented benchmark bug was silently fixed on two endpoints and left live
on a third, with no versioning or changelog — quantifying the benchmark-integrity
problem the project set out to audit. The two-axis auditor was decisive: the
desalted-InChIKey axis caught bbb's salt-form duplicates that Tanimoto could not
see, and correctly cleared ld50's collision as a false alarm. A single Tanimoto
threshold would have mis-called both.

## Method integrity
- No pipeline code was modified; every result came through the standard
  `Orchestrator` → `request_test()` path (one test read per endpoint).
- Validation stayed the headline everywhere; test reads are bounded one-shot
  confirmations. Several test points are favorable draws flagged by the
  valid-vs-test axis (dili +0.090 and caco2 −0.098 are the clearest), reinforcing
  that the flashy test number is never the claim.
